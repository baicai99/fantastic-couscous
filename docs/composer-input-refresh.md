# Composer 输入框改版说明（2026-03-07）

## 目标

- 参考截图对齐输入框形态：胶囊输入条、左侧 `+`、右侧发送主按钮。
- 保留既有发送、快捷选择器、动态提示词、高级变量等业务能力。
- 不引入语音能力，不调整页面整体背景主题。

## 交互映射

- 左侧上传入口：
  - 原“输入框上方上传行”迁移为输入条左侧 `+` 按钮。
  - 仍然触发同一隐藏文件输入控件，复用 `onSourceImagesAppend`。
  - 支持在输入框内 `Ctrl+V` 直接粘贴图片（仅拦截图片类型，文本粘贴保持原行为）。
  - 支持在聊天工作区任意位置直接拖拽图片上传（出现全局拖拽提示层，松手后加入参考图队列）。
- 中间输入区：
  - 继续使用 `Input.TextArea` 自动高度。
  - 普通 prompt 模式默认不展示占位文案；仅动态提示词模式展示简短模板示例占位（避免换行）。
  - 动态提示词 tag 在输入条上方轻微上移，减少遮挡输入区域。
  - 保留快捷触发与选择：
    - `/` 或 `、`：快捷功能项。
    - `@`：模型快捷补全。
    - `--`：参数命令补全。
  - Enter / Shift+Enter 行为保持不变。
- 右侧操作区：
  - 保留“高级变量”按钮与“发送”按钮。
  - 移动端将“高级变量”收敛为图标按钮，避免输入区挤压。

## 参考图区域

- 仍支持查看与单张删除。
- 展示方式调整为输入条下方紧凑面板，减少对主输入条视觉干扰。
- 当存在已上传图片时，输入框切换为“上图下输”布局：
  - 顶部显示大尺寸圆角缩略图（贴近截图中的预览块样式）。
  - 缩略图右上角使用悬浮圆形删除按钮。
  - 底部保留 `+`、输入区、高级变量与发送主按钮。
  - 整体容器高度增加，形成与截图一致的留白与视觉重心。
  - 预览图卡片的圆角与外层输入框改为同心圆关系，避免左上角圆弧错位。
  - 输入框自适应宽度计算已纳入左侧 `+` 按钮、右侧操作区、主行 gap、容器内边距与 TextArea 内边距，并增加安全余量（`20px`），避免“先换行再扩宽”。
  - 在未达到 `max-width` 前禁用自动换行（仅保留手动换行）；到达 `max-width` 后恢复自动换行。
  - 宽度更新增加“增长期只增不减”稳定策略：输入/换行等长度增长场景不触发收缩，删除内容时才允许回缩。
  - 删除回缩场景增加短延迟确认（`120ms`）后再收缩，避免“先缩后弹”的视觉抖动。
  - 多行输入时，左侧 `+` 与右侧发送按钮始终贴底对齐，不随输入区高度居中漂移。
  - 单行输入时，文本区保持垂直居中显示，避免出现光标/文本整体下沉。

## 消息区补充交互

- 当聊天列表未处于底部时，在屏幕固定位置（输入框上方中间区域）显示“向下箭头”悬浮按钮。
- 点击该按钮可平滑滚动到消息底部，快速回到最新消息。
- 按钮显示/隐藏带有 Q 弹过渡动画（弹入上浮 + 淡入 / 下沉淡出）。
